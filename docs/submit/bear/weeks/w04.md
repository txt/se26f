## Tests T10–T15

    T10 say("{name} has {hunger} hunger")(p) → correct string
    T11 say("{zzz}")(p) → prints warning, returns "<zzz?>"
    T12 say() returns a callable, not a string
    T13 now(hunger,-30)(p) with hunger=80 → p.hunger==50, returns 50
    T14 now(alive)(p) → p.alive==true, returns true
    T15 now(alive,0)(p) → p.alive==false

```lua
-- T10
local hunger = M.make("hunger")
local p = {name="Yogi", hunger=80}
M.say("{name} has {hunger} hunger")(p)
-- (capture stdout and assert in real harness)

-- T12
local fn = M.say("hello {name}")
assert(type(fn) == "function")

-- T13
local r = M.now(hunger,-30)(p)
assert(p.hunger == 50)
assert(r == 50)

-- T14
local alive = M.make("alive")
local r2 = M.now(alive)(p)
assert(p.alive == true)
assert(r2 == true)

-- T15
M.now(alive,0)(p)
assert(p.alive == false)
```

## TLAs

| # | TLA | Expansion | Definition | Bear example |
|---|-----|-----------|------------|--------------|
| 10 | CQS | Command-Query Separation | A function either changes state (command) or returns a value (query). Not both. Pick one; document it. | `now()` is a command: it mutates. `over()` is a query: it reads. They are never mixed. |
| 11 | SoC | Separation of Concerns | Different aspects of a problem should be handled by different code units with minimal overlap. | `say()` formats. `print()` outputs. `now()` mutates. Three concerns, three functions, zero overlap. |
| 12 | DP | Defensive Programming | Assume inputs may be wrong. Check, warn, and substitute rather than crash silently or noisily. | `say("{zzz}")` warns and substitutes `<zzz?>` rather than crashing or printing `nil`. |

## Anti-Patterns

| # | Name | Symptom | Consequence | Bear smell |
|---|------|---------|-------------|------------|
| 10 | CQ Violation | Function mutates state and also returns the mutated value as a query | Caller cannot tell if calling it twice is safe | `now()` that both sets `hunger` and returns the old value — which does the caller trust? |
| 11 | String Concatenation Spam | `"[" .. p.name .. "] hunger: " .. p.hp` copy-pasted with tiny variations | Noisy, typo-prone, untestable | The original `machine2.lua` — every action is a hand-rolled concatenation |
| 12 | Hidden Side-Effect | Function named as a query (`getHunger()`) that also mutates | Callers who read the name assume safety; bugs are invisible | A comparator `over()` that secretly logs or increments a counter |

## Python Idioms

| # | Idiom | Bad (newbie) | Good (idiomatic) |
|---|-------|--------------|-----------------|
| 10 | `str.format_map` | `msg.replace("{hunger}", str(p["hunger"]))` in a loop | `msg.format_map(p)` — one call, handles all keys, raises `KeyError` on missing |
| 11 | `dict.__missing__` | `if key not in d: return placeholder` before every access | Subclass dict, define `__missing__(self,k)` — called automatically on missing key, warn once |
| 12 | `@property` with setter validation | Direct attribute: `p.hunger = 150` (no range check) | `@hunger.setter def hunger(self,v): self._h = max(0,min(100,v))` — invariant enforced at assignment |
