# Week 8 — Two Machines (Cross-FSM Query)

## The Bear

Uncomment `-- S8` blocks. Add PlantFSM.

```
-- seed 42
-- tick 1
Plant#2 budding.
Bear#1  sniffs.  hunger=80
>> Bear#1 queries Plant#2: berries_ready=false
>> Bear#1: plant not ripe. foraging elsewhere. (attempt 1)

-- tick 2
Plant#2 RIPE!  yield=15
Bear#1  sniffs. hunger=85
>> Bear#1 queries Plant#2: berries_ready=true
>> Bear#1 eats! hunger=85 → 55
```

One machine queries another. They share nothing but the contract.

## Tests T25–T27

    T25 peer.get(token) returns correct live value
    T26 unstarted peer → token defaults false, no crash
    T27 dry_run detects A→B→A query cycle, prints warning

```lua
-- T25
local plant = make_plant("berry", 15, 2)
plant.berries_ready = true   -- manually ripen
assert(M.peer_get(plant, "berries_ready") == true)

-- T26
local unstarted = make_plant()
-- never called start()
local v = M.peer_get(unstarted, "berries_ready")
assert(v == false or v == nil)   -- graceful default

-- T27: cycle detection in dry_run
local a = make_bear(); local b = make_plant()
-- give a a query to b, and b a query back to a
a.peer = b; b.peer = a
local cycles = M.lint_cycles({a, b})
assert(#cycles > 0)
```

## TLAs

| # | TLA | Expansion | Definition | Bear example |
|---|-----|-----------|------------|--------------|
| 22 | LSP | Liskov Substitution Principle (peer level) | Any machine implementing `.get(token)` can replace any other. Tests use a mock; production uses PlantFSM. | `MockPlant` with hardcoded `get()` is a valid peer for testing `BearFSM` in isolation. |
| 23 | LI | Lazy Initialisation | Defer the cost of creation until first use. Document the default contract clearly. | Unstarted `PlantFSM.get(berries)` returns `false`. The default is documented, not accidental. |
| 24 | CBD | Contract-Based Design | Each module publishes a contract (interface). Callers depend on the contract, not the implementation. | `BearFSM` calls `peer.get(token)`. It does not care whether peer is a real plant, a mock, or another bear. |

## Anti-Patterns

| # | Name | Symptom | Consequence | Bear smell |
|---|------|---------|-------------|------------|
| 22 | Tight Coupling | `BearFSM` directly reads `plant.states.ripe.berries` | Change plant internals → bear breaks, even though contract is unchanged | Bear reaching into `plant.states["ripe"]["update"]` instead of calling `plant.get()` |
| 23 | Assume Peer Alive | `peer.berries_ready` accessed without nil-check | Crash on first tick when plant has not started yet | `if peer.berries_ready then` — nil-indexes on unstarted machine |
| 24 | Mock-Free Integration Test | Unit test spins up full ecosystem to test one bear decision | Slow, fragile, tests too many things at once | Running all 5 plants to test whether `BearFSM` correctly reads `berries_ready=true` |

## Python Idioms

| # | Idiom | Bad (newbie) | Good (idiomatic) |
|---|-------|--------------|-----------------|
| 22 | `unittest.mock.MagicMock` | Build a real `PlantFSM` just for bear tests | `mock_plant = MagicMock(); mock_plant.get.return_value = True` — plant never runs; bear is tested in isolation |
| 23 | `Protocol` (structural subtyping) | `class Peer(ABC): def get(self,t): ...` + explicit registration | `class PeerProtocol(Protocol): def get(self,t:str)->Any: ...` — any object with `.get()` qualifies; no inheritance |
| 24 | `getattr` with default | `if hasattr(peer,"berries_ready"): v=peer.berries_ready` | `v = getattr(peer, "berries_ready", False)` — one call, default built in, no double-lookup |
