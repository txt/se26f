# Week 5 — when{} Guards (Or-of-Ands)

## The Bear

Uncomment `-- S6` guard blocks.

```
-- seed 42  (hunger starts 90)
Bear#1 sniffs.  hunger=90
>> guard: hunger>70 AND found_food → TRUE → gorge
Bear#1 gorges! hunger=90 → 40

-- seed 7  (hunger starts 35)
Bear#1 sniffs.  hunger=35
>> guard: hunger>70 → FALSE
>> guard: hunger>30 → TRUE → nibble
Bear#1 nibbles. hunger=35 → 25

-- seed 13  (hunger starts 20)
Bear#1 sniffs.  hunger=20
>> all guards fail → sleep
Bear#1 sleeps.
```

Different seeds → different dietary choices. The FSM is alive.

## Tests T16–T18

    T16 first passing conjunct wins; second conjunct skipped
        even if it also passes
    T17 all conjuncts fail → eval_when returns nil
    T18 one false pred in a conjunct kills the whole conjunct

```lua
local hunger = M.make("hunger")
local found  = M.make("found")

-- conjunct that needs both: hunger>70 AND found==true
local gorge_clause  = {preds={M.over(hunger,70),
                               M.eq(found,true)},
                       target="gorge"}
local nibble_clause = {preds={M.over(hunger,30)},
                       target="nibble"}
local when_clauses  = {gorge_clause, nibble_clause}

-- T16: first conjunct wins
local p1 = {hunger=90, found=true}
assert(M.eval_when(when_clauses, p1) == "gorge")

-- T17: all fail
local p2 = {hunger=10, found=false}
assert(M.eval_when(when_clauses, p2) == nil)

-- T18: partial conjunct failure
local p3 = {hunger=90, found=false}  -- hunger ok, found fails
assert(M.eval_when(when_clauses, p3) == "nibble")
-- gorge fails (found=false), nibble passes
```

## TLAs

| # | TLA | Expansion | Definition | Bear example |
|---|-----|-----------|------------|--------------|
| 13 | LSP | Liskov Substitution Principle | Any implementation of an interface can replace any other without changing correctness. | `eval_when` takes any list of conjuncts. A mock `[{preds={}, target="eat"}]` is a valid guard. |
| 14 | ISP | Interface Segregation Principle | Don't force callers to depend on interfaces they don't use. Small interfaces beat fat ones. | States with no guard leave `guard=nil`. The engine checks for nil and skips. No empty stub required. |
| 15 | Fail-Fast | (universal term) | Detect errors at the earliest possible point; stop rather than propagate corrupt state. | `eval_when` stops at the first false pred in a conjunct (short-circuit). It does not evaluate the rest. |

## Anti-Patterns

| # | Name | Symptom | Consequence | Bear smell |
|---|------|---------|-------------|------------|
| 13 | Predicate Jungle | Deeply nested boolean logic inline in transitions | Untestable, unreadable, impossible to reorder | `if hunger>70 and found and not raining and tick%2==0 then` inside `run()` |
| 14 | Eager Evaluation Trap | All predicates evaluated before any short-circuit | Slow; crashes if later preds have preconditions | `all([p(agent) for p in preds])` — list comp evaluates everything; `all(p(a) for p in preds)` short-circuits |
| 15 | Fat Interface | Every state must implement guard, exit, update, trace, lint — even if nil | Cognitive overhead; stub errors | Requiring `guard={}` instead of `guard=nil` — forces empty value in every state |

## Python Idioms

| # | Idiom | Bad (newbie) | Good (idiomatic) |
|---|-------|--------------|-----------------|
| 13 | `all()` + generator | `ok=True; for p in preds: ok = ok and p(agent)` | `all(p(agent) for p in preds)` — short-circuits, one line, idiomatic |
| 14 | `any()` for or-of-ands | Nested for-loops with break flags | `any(all(p(ag) for p in c["preds"]) for c in clauses)` — one expression |
| 15 | Generator vs list comprehension | `all([pred(p) for pred in preds])` — evaluates all | `all(pred(p) for pred in preds)` — generator, short-circuits; drop the brackets |
