# Week 3 — Comparators (Pure Predicate Factory)

## The Bear

Uncomment `-- S3` blocks.

```
-- seed 42
Bear#1 sniffs.   hunger=80
>> hunger > 50: TRUE  → will eat
Bear#1 eats!

-- second run with hunger=30:
Bear#1 sniffs.   hunger=30
>> hunger > 50: FALSE → goes back to sleep
Bear#1 sleeps.
```

The same code, different data, different path.
Guards are data, not branches in the engine.

## Tests T7–T9

    T7  over(hunger,50) with hunger=80 → true; hunger=30 → false;
        hunger=50 → false (strictly greater)
    T8  to(energy,20) at 20 → true; at 21 → false
    T9  eq(found,true) with found=true → true; found=false → false

```lua
-- T7
local hunger = M.make("hunger")
local p = {hunger=80}
assert(M.over(hunger,50)(p) == true)
p.hunger = 30
assert(M.over(hunger,50)(p) == false)
p.hunger = 50
assert(M.over(hunger,50)(p) == false)  -- strictly >

-- T8
local energy = M.make("energy")
local q = {energy=20}
assert(M.to(energy,20)(q) == true)
q.energy = 21
assert(M.to(energy,20)(q) == false)

-- T9
local found = M.make("found")
assert(M.eq(found,true)({found=true})  == true)
assert(M.eq(found,true)({found=false}) == false)
```

## TLAs

| # | TLA | Expansion | Definition | Bear example |
|---|-----|-----------|------------|--------------|
| 7 | Pure Function | (universal term) | A function whose output depends only on its inputs, with no side-effects. Same input → same output, always. | `over(hunger,50)(p)` reads `p.hunger`, returns bool, touches nothing else. Safe to call 100 times. |
| 8 | CoI | Composition over Inheritance | Prefer assembling behaviour from small composable pieces over building class hierarchies. | A guard is a list of predicates. No `Predicate` base class. Compose `over`, `to`, `eq` with plain tables. |
| 9 | LoD | Law of Demeter (Don't Talk to Strangers) | A function should only call methods on its direct arguments, not on objects returned by those arguments. | The engine calls `pred(payload)`. It does not reach into `pred.threshold`. It knows only the callable interface. |

## Anti-Patterns

| # | Name | Symptom | Consequence | Bear smell |
|---|------|---------|-------------|------------|
| 7 | Boolean Blindness | Functions return raw `true`/`false` with no context | Caller cannot tell which predicate failed or why | `check(p)` returns false — was it hunger? energy? found_food? |
| 8 | Stringly Typed | Logic encoded as strings: `if pred=="hungry"` | Typo creates a silent no-op; no static check catches it | `transitions = { hungry="eat" }` where event is actually `"hunger"` |
| 9 | Arrow Anti-Pattern | Deeply nested if/elseif chains for predicate logic | Unreadable, untestable, impossible to reorder | `if hunger>50 then if energy>20 then if found then ...` — 4 levels deep |

## Python Idioms

| # | Idiom | Bad (newbie) | Good (idiomatic) |
|---|-------|--------------|-----------------|
| 7 | `functools.partial` | `lambda p: p["hunger"] > 50` for every threshold | `over = lambda tok,n: partial(operator.gt, n)` — one factory, operator module does the comparison |
| 8 | `operator` module | `lambda a,b: a > b` reinvented everywhere | `import operator; op.gt(a,b)` — stdlib, named, serialisable |
| 9 | `Callable[[P],bool]` type alias | No type hints on predicates | `Pred = Callable[[dict], bool]` — names the concept; `mypy` enforces it |
