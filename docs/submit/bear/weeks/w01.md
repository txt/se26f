# Week 1 — Bare Engine

## The Bear

```
lua bear.lua 42
-- seed 42  (replay: lua bear.lua 42)
=== S1: one bear ===
Yogi sleeps.
Yogi sniffs around.
Yogi eats!
```

Three states. Five-line engine. Queue drives it.
This is the whole system — everything later is elaboration.

## Tests T1–T3

    T1  queue ["wake","smell","done"] → states fire in order
    T2  empty queue on entry → payload returned, no crash
    T3  unknown event "quake" → stays in current state silently

```lua
-- T1
local b = make_bear("Test", 80, 100)
b.queue = {"wake","smell","done"}
local r = M.start(b)
assert(r.current == "sleep")   -- cycled back

-- T2
local b2 = make_bear()
b2.queue = {}
local r2 = M.start(b2)
assert(r2 ~= nil)

-- T3
local b3 = make_bear()
b3.queue = {"quake"}
local r3 = M.start(b3)
assert(r3.current == "sleep")  -- stayed put
```

## TLAs

| # | TLA | Expansion | Definition | Bear example |
|---|-----|-----------|------------|--------------|
| 1 | SSOT | Single Source of Truth | Every fact lives in exactly one place. All other references are lookups. | `make_bear` is the one place bear structure is defined. No other code duplicates it. |
| 2 | YAGNI | You Aren't Gonna Need It | Don't build features before they are required by a failing test. | The engine is 5 lines. Guards, lint, trace are all commented out. They don't exist yet. |
| 3 | SRP | Single Responsibility Principle | Each function does exactly one thing. If you need "and" to describe it, split it. | `run()` only advances the machine. It does not print, log, or validate. |

## Anti-Patterns

| # | Name | Symptom | Consequence | Bear smell |
|---|------|---------|-------------|------------|
| 1 | Magic String Soup | State names like `"s1"`, `"st_a"` scattered as literals | One rename breaks silently in 4 places | Writing `"slep"` instead of `"sleep"` — no error, wrong behaviour |
| 2 | God Function | One function does parse, validate, run, and print | Untestable; every change risks everything | A `run()` that also lints and traces is a god function waiting to happen |
| 3 | Eyeball Verification | "I ran it and it looked right" is the test suite | Regressions invisible; bugs hide in untested paths | Watching the bear print is fine for week 1. It is not fine for week 5. |

## Python Idioms

| # | Idiom | Bad (newbie) | Good (idiomatic) |
|---|-------|--------------|-----------------|
| 1 | `@dataclass` + `__repr__` | Write `__init__`, `__repr__`, `__eq__` by hand | `@dataclass class Bear: name:str; hunger:int=80` — compiler writes the boilerplate |
| 2 | `dict` as namespace | Parallel lists: `names=[]`, `hungers=[]` | `agent = {"name":"Yogi","hunger":80}` — one dict, one agent, one place |
| 3 | `sys.argv` + `os.getenv` priority chain | Hardcode the seed | `seed = int(sys.argv[1]) if len(sys.argv)>1 else int(os.getenv("SEED","0")) or int(time.time())` |
