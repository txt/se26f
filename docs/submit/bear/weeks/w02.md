# Week 2 — Tokens (SSOT Registry)

## The Bear

Uncomment `-- S2` blocks in fsm.lua and bear.lua.

```
-- seed 42
=== S2: tokens ===
Bear#1 sleeps.   hunger=80 energy=100
Bear#1 sniffs.   hunger=80 energy=100
Bear#1 eats!     hunger=80 energy=100
-- token registry: hunger_1(id=2) energy_1(id=3)
```

Tokens now have unique ids and appear in the payload dump.
Duplicate registration warns loudly.

## Tests T4–T6

    T4  make("hunger") twice → second call prints warning,
        returns same token (no duplicate id)
    T5  two different names → get different ids, both in registry
    T6  dry_run=true → prints all token names and current values

```lua
-- T4
local t1 = M.make("hunger")
local t2 = M.make("hunger")   -- should warn
assert(t1.id == t2.id)

-- T5
local ta = M.make("speed")
local tb = M.make("weight")
assert(ta.id ~= tb.id)

-- T6
local b = make_bear("Test", 60, 90)
M.dry_run(b)   -- prints token table, no transitions fired
```

## TLAs

| # | TLA | Expansion | Definition | Bear example |
|---|-----|-----------|------------|--------------|
| 4 | DRY | Don't Repeat Yourself | Every piece of knowledge has one authoritative representation. Duplication is a maintenance liability. | `make("hunger")` called once. The id is never typed again. All comparators refer to the token object, not the string. |
| 5 | OCP | Open/Closed Principle | A module is open for extension but closed for modification. Add new behaviour without editing existing code. | Adding `make("speed")` requires zero engine changes. The registry extends; `run()` is untouched. |
| 6 | Meaningful Names | (not an acronym — a discipline) | Names should reveal intent. The reader should not need a comment to understand what a variable holds. | `hunger` beats `t3`. `make_bear` beats `new_agent`. `new_id` beats `inc`. |

## Anti-Patterns

| # | Name | Symptom | Consequence | Bear smell |
|---|------|---------|-------------|------------|
| 4 | Primitive Obsession | Using raw ints/strings where a named type would clarify | `rules[3]` instead of `rules[hunger.name]`; meaning lost | Passing token id `3` to comparators instead of the token object |
| 5 | Silent Duplicate | Second `make("hunger")` quietly overwrites first | All reads of the first token now return wrong data | Plant and bear both call `make("hunger")` — now they share state by accident |
| 6 | Implicit Registry | Global state modified without a named registration step | Impossible to audit what tokens exist; dry_run cannot list them | Creating token fields directly in the payload dict, bypassing `make()` |

## Python Idioms

| # | Idiom | Bad (newbie) | Good (idiomatic) |
|---|-------|--------------|-----------------|
| 4 | `Enum` for ids | `HUNGER = 1; ENERGY = 2` as bare ints | `class Token(Enum): HUNGER=auto(); ENERGY=auto()` — singleton, comparable by identity, self-documenting |
| 5 | `dict` comprehension as registry | `for t in tokens: reg[t.name] = t` (for-loop + assign) | `registry = {t.name: t for t in tokens}` — one expression, same semantics |
| 6 | `__repr__` on dataclass | Manually write `def __repr__(self): return f"Token({self.name})"` | `@dataclass` generates it; add `repr=True` (default) and it's free |
