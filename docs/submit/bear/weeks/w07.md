# Week 7 — Lint (Static Analysis)

## The Bear

Uncomment `-- S7` lint blocks. Introduce a deliberate typo.

```
-- seed 42
=== LINT: Bear#1 ===
LINT [Bear#1]: ghost state 'slep' from 'sniff'
LINT [Bear#1]: dead end 'gorge' (no outbound transitions)
=== 2 warnings. Fix before running. ===

-- after fixing typo:
=== LINT: Bear#1 ===
=== 0 warnings. OK. ===
Bear#1 sleeps.  hunger=80
...
```

Bugs caught before the first tick. Static analysis is free testing.

## Tests T22–T24

    T22 clean machine → zero lint warnings
    T23 ghost state 'slep' detected and named
    T24 dead-end non-terminal state detected

```lua
-- T22: clean machine
local clean = make_bear("Clean", 80, 100)
local warns = M.lint_collect(clean)  -- returns list, not print
assert(#warns == 0)

-- T23: inject ghost state
local ghost_bear = make_bear("Ghost", 80, 100)
ghost_bear.states.sniff.transitions.smell = "slep"  -- typo
local w2 = M.lint_collect(ghost_bear)
local found = false
for _,w in ipairs(w2) do
  if w:find("ghost") and w:find("slep") then found=true end end
assert(found)

-- T24: dead-end state
local dead_bear = make_bear("Dead", 80, 100)
dead_bear.states.gorge = {
  action=function(p) end, transitions={} }
-- gorge is reachable but has no outbound, not terminal
dead_bear.states.eat.transitions.done = "gorge"
local w3 = M.lint_collect(dead_bear)
local found2 = false
for _,w in ipairs(w3) do
  if w:find("dead end") and w:find("gorge") then found2=true end end
assert(found2)
```

## TLAs

| # | TLA | Expansion | Definition | Bear example |
|---|-----|-----------|------------|--------------|
| 19 | SA | Static Analysis | Examining code or data for errors without executing it. Catches whole classes of bugs at zero runtime cost. | `lint()` walks the rules table before `start()`. Typos in state names caught before first tick. |
| 20 | SVE | Separation of Validation from Execution | Validation and execution are distinct passes. Each is testable in isolation. | `lint(agent)` reads rules and prints. It does not run the machine. You can call lint without ever calling start. |
| 21 | FiF | Fail-Fast (at the latest: load time) | The earlier an error is detected, the cheaper it is to fix. Load-time > first-run > deep-run. | A ghost state caught at lint costs 1 second. The same bug caught at tick 500 of a 1000-tick run costs an hour. |

## Anti-Patterns

| # | Name | Symptom | Consequence | Bear smell |
|---|------|---------|-------------|------------|
| 19 | Linting as Afterthought | Tests exist; lint does not. Typos discovered at runtime. | A one-character typo produces a silent wrong-path bug that takes hours to find | `"slep"` in transitions silently keeps the bear in the current state forever |
| 20 | Print-and-Eyeball (formalised) | Lint output goes to stdout with no structured return value | Cannot assert on lint results in tests; cannot count warnings | `lint()` that only prints — you cannot write `assert(#warnings == 0)` |
| 21 | Swallowed Warning | `if unknown_state then -- ignore` | Errors vanish; codebase accumulates silent rot | Transition to unknown state silently treated as stay-put, no record kept |

## Python Idioms

| # | Idiom | Bad (newbie) | Good (idiomatic) |
|---|-------|--------------|-----------------|
| 19 | `warnings.warn` | `print(f"WARNING: {msg}")` for lint output | `import warnings; warnings.warn(msg, UserWarning)` — controllable severity, filterable, capturable by `pytest.warns()` |
| 20 | `pytest.mark.parametrize` | 24 nearly identical test functions | `@pytest.mark.parametrize("state,event,expected", cases)` — one function, N cases, one failure report per case |
| 21 | Collecting vs printing | Lint function prints directly | `def lint(agent) -> list[str]: warns=[]; ...; return warns` — caller decides what to do; testable with `assert len(lint(a))==0` |
